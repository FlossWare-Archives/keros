Keros @keros-project.release.VERSION@

Requirements

Project Requirements

  Keros must:
  • Support all platforms that support Java and Ant.
  • Work for you not against you - even "right out of the box."
  • Provide enough default functionality to be "just good enough."
  • Be project friendly - contain enough functionality to use for whatever is needed for a project.
  • If "just good enough" is not sufficient - be easy to override in a hassle free way.
To understand the source, one should be familiar with Ant, XML (including namespaces), XSL, Ant-Contrib, and Beanshell.

Downloads

Manual Downloads
  • Ant 1.7.1+ (preferably 1.7.1 to simplify using the included libraries)
  • Java 1.5+
  • Subversion:  only if used for source control and utilizing the Subversion to HTML macrodef.  It convert Subversion logs to a decent HTML representation.
Included
  • Ivy:  manages third party dependencies (or libraries).
  • Find Bugs:  Java code analysis.  Please note:  at the time of this release, we were having trouble with utilizing Ivy to download this library.
Both Ivy and Find Bugs are included, and therefore available to those projects that import either keros.xml or keros-project.xml.  Also available are the automagic downloads used by Keros.

Automagic
The following third party libraries will be used by Keros (and therefore available to you):
  • Ant-Contrib:  useful Ant libraries.
  • Beanshell:  used in some scriptdef definitions.
  • Bean Scripting Framework:  used in some scriptdef definitions.
  • Castor:  used to generate Java from XSD's.
  • JavaCC:  used for building parsers.
Keros (actually Ivy) downloads and installs the aforementioned automagic libraries to ${user.home}/.keros/[release]/lib - for example /home/sfloess/.keros/1.0/lib
