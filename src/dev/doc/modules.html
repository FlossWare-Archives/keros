Project Modules

  Keros is divided into the following modules:

Ant

The Ant modules are divided into logical file types as follows:
  • Build:  Main build script for Keros.
  • Keros core functionality:  Contains reusable macrodefs/properties - useful for most projects.  Only one target is defined:  it prints properties related to this file.
  • Keros Project:  Contains Ant tasks for useful development related features such as compiling, building releases, etc.  The contents of this file are probably "good enough" for most development environments.  Keros itself uses this file internally for building releases and testing.

Resources

This module contains internationalized messages used by the above Ant modules and are divided into sub-directories by locale (for example en_US for the U.S. locale).  Resource files are named like keros*-messages.xml - for example keros-messages.xml or keros-project-messages.xml.

XSLT

This module contains reusable XSLT's as follows:
  • ant2html.xsl:  An XSLT capable of converting an Ant script into a meaningful HTML representation.  The macro ant2html generates HTML against the aforementioned Ant modules defined above.
  • svn2html.xsl:  An XSLT capable of converting a Subversion log history into a meaning HTML representation.  Please note:  to use this XSLT, it is assumed an XML log file is generated from Subversion similar to:
svn log [dir] --xml -r [revision start]:[revision end] -v
The macro svn2html uses the macro svn-xml-log to generate the Subversion log for this release.
