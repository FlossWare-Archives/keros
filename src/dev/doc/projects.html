Projects

Starting with Keros 1.3, the concept of project has been decoupled from the now defunct keros-project.xml and encapsulated into separate files that provide proper separation of concerns.  The various project types are listed below:

keros-base-project.xml

This is a slimmed down version of keros-project.xml and is imported by all project files.  This script defines some common project properties, such as directories, as well as the following project lifecycle targets:
  1. keros-base-project.all:  this is the default target depending on the subsequent targets that follow (and in the order listed below).
  2. keros-base-project.clean:  removes the directory ${keros-base-project.generated.HOME}.
  3. keros-base-project.generate:  empty target.  Should be overridden when some form of generation is to occur (for example JavaCC).
  4. keros-base-project.compile:  empty target.  Should be overriden when something is to be compiled (for example javac).
  5. keros-base-project.test:  empty target.  Should be overriden when some form of testing is to occur (for example JUnit).
  6. keros-base-project.document:  empty target.  Should be overridden when documentation is to be emitted (for example Javadoc, converting Subversion logs to HTML or an Ant script to HTML).
  7. keros-base-project.analyze:  empty target.  Should be overriden when some form of analysis should occur (for example FindBugs).
  8. keros-base-project.sub-projects:  empty target.  Should be overridden when sub-projects are to be processed.
  9. keros-base-project.release:  empty target.  should be overridden for appropriate release types.
Please note:  in most instances, the project types listed below will, when appropriate, override targets listed above as "should be overridden."
